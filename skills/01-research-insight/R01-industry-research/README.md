# 行业研究（industry-research）

| 项目 | 内容 |
|------|------|
| 中文名称 | 行业研究 |
| 英文名称 | industry-research |
| Skill 编号 | R01 |
| 当前版本 | 1.3.0 |
| 一句话介绍 | 系统性研究行业格局、产业链、玩家、趋势与机会/风险，不虚构量化数据。 |

## 解决的问题

进入一个新行业 / 赛道前缺乏系统判断；拿不准"这个赛道还能不能进"；写行业方案或投资备忘录缺少结构化的行业结论。

## 适合人群

新媒体运营总监 / 操盘手、品牌与市场负责人、内容创业者、需要做行业竞调或战略分析的人。

## 适用场景

- 评估是否进入某行业 / 赛道（如短视频、直播电商、知识付费、私域）。
- 寻找市场机会或做行业竞调。
- 为方案、立项或投资判断提供行业背景。

## 不适用场景

- 只分析单个竞品的定位、价值主张 → 用 **竞品定位分析（R02）**。
- 只分析竞品的内容方向、栏目、形式 → 用 **竞品内容分析（R03）**。
- 只监测近期热点与社会趋势 → 用 **热点趋势监测（R07）**。
- 只识别负面舆情与升级风险 → 用 **舆情风险识别（R08）**。

## 核心能力

界定行业边界与口径；拆解产业链 / 价值链；刻画玩家格局与集中度；提炼趋势驱动因素；列出机会与风险（带严重度与依据）；给出下一步建议。

## 必要输入

1. 行业 / 赛道名称与范围（越具体越好，如"美妆直播电商"而非泛泛"电商"）。
2. 研究目的（评估进入 / 找机会 / 写方案 / 竞调）。
3. 地理市场（默认中国大陆；出海需说明）。
4. 时间范围（默认近 2–3 年 + 未来 1–2 年预判）。

缺失且可合理假设时，会标注"假设：…"后继续；范围完全空白会集中询问一次。

## 主要输出

Markdown 报告，至少包含：行业一句话定义与边界、产业链结构、市场规模与量级（无可靠依据则明确不输出量级）、玩家格局与集中度、趋势驱动因素、机会清单 + 风险清单、下一步建议、数据周期与检索日期。

## 简短使用示例

用户说："研究一下职场知识付费赛道，我想做新人方向。"

输出：界定边界（职场技能方向）→ 产业链（内容生产-平台分发-私域转化）→ 规模（无可靠依据，明确"现有资料不足，无法可靠判断市场规模。"）→ 玩家（头部平台 / 腰部机构 / 独立创作者）→ 趋势（AI 提效、垂直化，标"推断"）→ 机会（职场新人细分，按门槛 / 空间排序并标依据）+ 风险（复购可能偏低，仅依据行业常识、无直接样本证据，置信度低，标"待验证"）→ 建议（先验证选题再投入）。

## 与相邻 Skill 的区别

R01 看**整个行业**的格局与机会；R02 只看**单个竞品**的定位；R03 只看竞品的**内容**；R07 看**近期热点**；R08 看**舆情风险**。研究方向不同，不互相替代。

## 事实、推断与未知的处理规则

三级标注：

- **已观察事实**：有来源 / 数据支撑。
- **证据支持的推断**：标"推断"。
- **未知**：标"待验证"。

来源优先级：政府 / 监管机构 / 统计机构 / 行业协会 / 企业财报 > 权威研究报告 > 主流媒体 > 企业宣传 > 社交讨论（低优先级须降级标注）。不把推断写成事实，无来源数字不输出。

## 数据、来源或样本限制

市场规模仅在有可靠来源或明确、可解释、可复核的估算方法时才输出，并标注来源、周期与口径；无可靠依据不输出任何市场量级。小样本不输出伪精确比例，改用"少量样本 / 频次信号"。引用数据须标数据周期与检索日期。

## 隐私与安全边界

未经用户授权不调用付费数据接口；用户已授权并配置的工具（搜索 / 浏览器 / 平台数据 / 文件分析）可以使用。分析材料仅用于本次研究，不做无关留存。
