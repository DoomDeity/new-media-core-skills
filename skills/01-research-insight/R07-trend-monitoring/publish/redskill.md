- **一句话介绍（30 字内）**：判断热点处在什么阶段、值不值得跟、与你业务的关系。

## 主要解决的问题

想跟热点却不知道处在什么阶段、值不值得跟、和自己业务有什么关系；容易把旧知识当当前趋势、把单条爆款当趋势、把跨平台扩散当成必要条件。

## 3—5 个典型使用场景

1. 评估某话题是否值得跟进、处在什么阶段。
2. 判断近期行业 / 平台 / 内容 / 社会变化信号。
3. 判断热点与自身业务的相关性和持续性。
4. 涉及"当前 / 最新"话题时主动核验，不靠旧记忆下结论。

## 适合人群

新媒体运营、内容策划、品牌与增长团队、需要做热点跟进判断的人。

## 推荐调用语句

"AI 求职是不是趋势，现在值不值得跟？"
"最近行业有什么变化信号，和我业务有关系吗？"
"这个热点处在什么阶段了，要不要跟进？"

## 输入要求

关注领域 / 平台 / 业务背景、具体话题或信号（可选，不提供则广谱扫描）、时间窗口（仅口径参考）、用户业务与目标。涉及"当前 / 最近"须提供可检索条件或授权实时工具。

## 输出内容

时间快照、监测口径、趋势 / 热点清单（信号 / 来源字段 / 热度阶段 / 相关性 / 持续性 / 状态标注）、分类视图（短期热点 / 正在形成趋势 / 已稳定变化 / 噪声）、相关性排序、跟进建议（不替你拍板）。

## 不适用场景

行业长期结构研究（归 R01）、平台规则变化本身（归 P05）、负面舆情风险（归 R08）、只归纳评论痛点（归 R05）、直接生成选题内容（归 C 类）。

## 事实和数据边界

当前热点必须主动检索最新资料，不默认依赖训练记忆；引用标来源与日期；同源转载只算扩散不倍数；单平台内多独立持续信号也可成趋势（跨平台非必要）；不虚构搜索量 / 热度指数；单一时间快照不足判高峰或回落，优先"无法判断"；无法联网明确报告无法验证当前状态。

## 隐私及合规提醒

含私信 / 个人信息先匿名化不提交外部工具；未经授权不调用付费数据接口；实时工具不可用时只分析有日期的已提供材料。

## 发布文案（适合小红书 / RedSkill 介绍页）

追热点最怕两件事：一是把一条爆款当成大趋势，二是拿去年的旧梗当本月热点。这个热点趋势监测 Skill 帮你把"该不该跟"变成可核对判断：先写明时间快照（截至 / 检索 / 窗口），再追溯原始来源、区分原创和转载扩散，按"起步 / 升温 / 高峰 / 回落 / 稳定 / 无法判断"给热度阶段并附依据，最后判断相关性和持续性、给你跟进建议——但绝不替你拍板。它不会虚构热度指数，遇到"当前 / 最新"会主动检索；实时工具不可用时，会老实告诉你"无法验证当前状态"，而不是拿训练记忆编一个。适合运营和策划。
