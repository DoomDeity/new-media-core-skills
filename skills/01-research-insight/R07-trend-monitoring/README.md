# 热点趋势监测（trend-monitoring）

| 项目 | 内容 |
|------|------|
| 中文名称 | 热点趋势监测 |
| 英文名称 | trend-monitoring |
| Skill 编号 | R07 |
| 当前版本 | 1.0.0 |
| 一句话介绍 | 监测近期行业 / 平台 / 社会热点信号，判断热度阶段、相关性与持续性，无法联网时明确报告无法验证当前状态。 |

## 解决的问题

想跟热点却不知道话题处在什么阶段、值不值得跟、和自己业务有什么关系；容易把旧知识当当前趋势、把单条爆款当趋势、把跨平台扩散当成趋势的必要条件；涉及"当前 / 最新"却默认依赖训练记忆。

## 适合人群

新媒体运营、内容策划、品牌与增长团队、需要做热点跟进判断的人。

## 适用场景

- 评估某话题是否值得跟进、处在什么阶段、和自己业务有什么关系。
- 判断近期行业 / 平台 / 内容 / 社会层面的变化信号。
- 判断热点的相关性、持续性与跟进价值。

## 不适用场景

- 系统研究行业长期结构、产业链、市场规模、长期驱动因素与系统机会（"这个赛道还能不能进""行业格局与玩家"） → 用 `industry-research`（R01）。路由以任务目的为主：要识别近期变化、热度阶段、持续性、跟进价值归 R07；要研究行业结构、产业链、市场机会归 R01。
- 监测平台规则、政策条款、功能机制变化本身 → 用 `platform-rule-monitoring`（P05）。若问"这个规则变化是否正在形成行业趋势、对业务有什么影响"，R07 可在 P05 事实基础上继续判断后续影响。
- 识别负面舆情、争议议题与升级风险 → 用 `public-opinion-risk-identification`（R08）。
- 只归纳一批评论的痛点与情绪 → 用 `user-comment-analysis`（R05）。
- 直接生成选题或内容 → 用 `topic-generation`（C04）等。

## 核心能力

时间快照（每条研判写明截至时间 / 时区 / 检索时间 / 各信号发布日期 / 覆盖窗口）；来源独立性与去重（多家媒体转载同一原始消息只算一个原始信号 + 若干扩散信号）；热度阶段定性（起步 / 升温 / 高峰 / 回落 / 稳定 / 无法判断，单一时间快照优先"无法判断"）；相关性与持续性判断；分类输出（短期热点 / 正在形成的趋势 / 已趋于稳定的变化 / 噪声或证据不足）；相关性排序；不替用户决定是否追热点。

## 必要输入

1. 关注领域 / 平台 / 业务背景（如"小红书职场内容""抖音本地生活"）。
2. 具体想核验的话题或信号（可选；不提供则做广谱扫描）。
3. 时间窗口（仅作分析口径参考，非路由依据）。
4. 用户业务与目标（用于判断相关性）。

缺失且可合理假设时，标注"假设：默认面向新媒体运营者"后继续；若完全无关注领域，集中询问一次（仅问关注哪个行业 / 平台）。

## 主要输出

Markdown 监测简报，至少包含：时间快照、监测口径、趋势 / 热点清单（每条含观察到的信号、来源字段、热度阶段、相关性、持续性、状态标注）、分类视图、相关性排序（定性 + 依据）、给用户的建议（标「建议」，不替用户拍板）、明确"结论基于检索到的公开信号，热度阶段与持续性含推断，需持续观察；若实时工具不可用，已注明无法验证当前状态"。

## 简短使用示例

用户说："AI 求职是不是趋势？"（时间快照：分析截至 2026-07-20 北京时区，检索 2026-07-20，覆盖窗口 30 天）。

输出：信号（垂直社区内多独立创作者持续讨论 AI 简历工具，标事实）→ 热度阶段（升温，依据前 15 天新增 2 个独立创作者信号、后 15 天新增 4 个并首次出现 1 个独立媒体原创信号与 1 个平台官方功能信号）→ 相关性（高，受众为求职者）→ 持续性（待验证，仅 30 天窗口，升温不等于中期持续性）→ 分类（正在形成的趋势，平台内且开始跨主体扩散）；另有一条单账号爆款视频点击高但无后续讨论，归类"噪声或证据不足"；建议（先做 1–2 篇测评内容验证受众反应，不盲目 all-in）。

## 与相邻 Skill 的区别

R01 看行业长期结构与市场机会，R07 看近期变化与热度阶段；P05 看平台规则变化本身，R07 在 P05 事实基础上看其引发的趋势与影响；R08 看负面舆情与升级风险，R07 只做中性趋势判断。研究方向不同，不互相替代。

## 事实、推断和未知的处理规则

三级标注：

- **事实（已观察信号）**：有来源 / 日期支撑的可见信号。
- **推断（趋势与持续性判断）**：标"推断"，须说明依据。
- **未知（待验证）**：证据不足暂不能判断的部分。

不虚构搜索量、热度指数、增长率、参与人数等无来源数字；无可靠口径明确"无法量化热度"。

## 数据、来源或样本限制

实时优先：当前热点必须主动检索最新公开资料，不默认依赖训练记忆；引用须标来源与日期。来源独立性：追溯原始发布者 / 官方 / 首发；转载视为扩散而非独立信号；独立来源数量与转载关系须说明；搜索结果数量不代表热度。证据门槛与跨平台：单个平台或垂直社区内出现多个独立、持续、重复信号，也可构成平台内趋势（跨平台扩散属于增强证据，不是必要条件）；单条爆款内容、单个媒体报道或同源转载仍不足以构成趋势。样本说明：信号数量、独立来源数、覆盖平台、周期须标注；小样本不输出伪精确比例。

## 隐私与安全边界

若材料涉及用户私信 / 个人信息，先匿名化，不提交未处理隐私材料到外部工具。涉及"当前 / 最近 / 最新 / 正在发生"任务必须使用当前可用的搜索、浏览器、平台数据或其他实时工具；若实时工具不可用、检索失败或只能获得陈旧资料，不得给出确定的当前趋势结论，应明确报告"无法验证当前状态"，只分析用户已提供且有日期的材料。
