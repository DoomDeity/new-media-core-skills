## 主要解决的问题

有一批评论 / 弹幕 / 私信，想提炼痛点信号、需求推断、情绪和高频问题；做内容复盘、产品优化缺归纳。

## 3—5 个典型使用场景

1. 归纳评论区在说什么、情绪怎么样。
2. 从评论提炼痛点信号与需求推断。
3. 统计高频问题与意图分布。
4. 为内容复盘、产品优化、选题灵感提供输入。

## 适合人群

新媒体运营、内容创作者、产品经理、用户运营、需要做评论复盘的人。

## 推荐调用语句

"看看这些评论在说什么？"
"从评论里提炼需求。"
"评论区情绪怎么样？"

## 输入要求

评论文本（原始粘贴或整理列表）、可选上下文（内容主题 / 平台 / 目的 / 周期）。默认只分析你提供的材料。

## 输出内容

样本概况（含匿名化说明）、情绪分布、评论意图分布（与情绪独立）、痛点信号与需求推断、高频问题 TOP、行动建议。

## 不适用场景

主动挖掘需求（归 R06）、识别舆情风险（归 R08）、写单条公开回复（归 P03）、制定负面评论处理方案（归 P04）。

## 事实和数据边界

情绪与意图属"推断"，须标注基于样本，不夸大代表性。痛点 / 需求非确证事实，标"推断"。小样本用实际数量，不输出伪精确比例。引用代表评论须来自用户实际输入并继续匿名化，不得编造。

## 隐私及合规提醒（重要）

含个人姓名、联系方式、账号 ID 等可识别信息的评论，**先匿名化**再分析。未经匿名化，不得将含真实姓名、手机号、邮箱、账号 ID、订单号、地址或私信可识别信息的原始材料提交外部搜索或第三方工具。默认只分析你提供的材料；仅当你明确要求补充公开背景时才外联，且涉及个人信息的材料仍须先匿名化。请勿在输入中直接粘贴未脱敏的个人隐私数据。

## 发布文案（适合小红书 / RedSkill 介绍页）

评论区是最好的用户调研。这个用户评论分析 Skill 帮你把一堆评论变成结构化结论：情绪分布、意图分类、痛点信号、高频问题，重复评论还会保留成频次信号（毕竟"怎么买"出现 15 次和出现 1 次完全不同）。它默认只分析你给的材料、不瞎联网，而且把"推断"和"事实"分清楚——需求是推断，不是你实打实的订单。最重要的一点：带个人信息的评论它会先匿名化，不会拿去乱搜。适合要做评论复盘、找选题灵感的同学。
