# 竞品营销策略拆解（competitor-marketing-strategy-analysis）

| 项目 | 内容 |
|------|------|
| 中文名称 | 竞品营销策略拆解 |
| 英文名称 | competitor-marketing-strategy-analysis |
| Skill 编号 | R04 |
| 当前版本 | 1.3.0 |
| 一句话介绍 | 拆解竞品传播、获客、承接与转化路径，不推测投放占比或成交数据。 |

## 解决的问题

想弄清竞品怎么获客、公域到私域怎么走、转化链路如何；做增长对标、转化链路参考、钩子与漏斗研究时缺方法。

## 适合人群

新媒体运营总监 / 操盘手、增长负责人、内容创作者、需要做竞品增长对标的人。

## 适用场景

- 拆解竞品怎么传播、从哪获客、私域怎么承接、怎么转化成交。
- 做增长对标、转化链路参考、钩子与漏斗研究。

## 不适用场景

- 分析竞品定位 / 价值主张 → 用 **竞品定位分析（R02）**。
- 分析竞品内容栏目与形式 → 用 **竞品内容分析（R03）**。
- 诊断"自己"的转化漏斗 → 用 **转化漏斗诊断（G02）**。
- 设计"自己"的内容获客路径 → 用 **内容获客路径（G01）**。

## 核心能力

绘制传播矩阵；识别可观察获客渠道（按证据强弱标注）；还原承接方式；梳理转化路径（逐节点标已观察 / 推断 / 未知）；提取关键钩子；总结可借鉴策略与无法复制的外部条件。

## 必要输入

1. 竞品名称与可观察链路（公域账号、落地页、社群、商品页、私信话术）。
2. 分析重点：传播 / 获客 / 承接 / 转化（可多选，默认全链路）。

链路不全时基于可见部分标「已观察 / 推断 / 未知」；连公域入口都不明则集中询问一次。

## 主要输出

Markdown 拆解，至少包含：传播矩阵（平台 → 角色）、可观察获客渠道与证据强弱（不写占比）、承接方式（钩子 + 入口 + 证据标注）、转化路径（阶段 + 关键动作，逐节点标已观察 / 推断 / 未知）、关键钩子清单、可借鉴策略（建议）、数据周期与检索日期。

## 简短使用示例

用户说："拆解某知识付费商家的抖音引流私域路径。"

输出：传播（抖音短视频种草，已观察）→ 获客渠道（评论区留资、主页链接，已观察；付费投放，未知）→ 承接（企业微信 + 资料包钩子，已观察）→ 转化路径（主页入口：已观察；9.9 元体验产品：已观察；体验产品是否主要导向正价产品：推断；实际成交路径与转化率：未知）→ 钩子（免费 SOP 资料）→ 借鉴（用资料包做留资钩子，但需自有产品承接）。

## 与相邻 Skill 的区别

R04 拆竞品的**营销获客转化路径**；R02 做竞品**定位**；R03 做竞品**内容**；G01 设计"自己"的获客路径；G02 诊断"自己"的漏斗。对象与目的不同。

## 事实、推断与未知的处理规则

三级标注：已观察事实 / 证据支持的推断（标"推断"）/ 未知（标"待验证"）。对投放占比、转化率、成交数据一律标"未知 / 待验证"，不推测。来源优先级：竞品官方账号、官网、产品页、落地页、公开活动页和用户一手材料 > 可观察内容与数据 > 权威第三方研究 > 主流媒体 > 社交讨论。

## 数据、来源或样本限制

内部投放占比、渠道贡献比例、真实转化率、成交额、ROI 属竞品机密或无法公开观察，一律标"未知 / 待验证"，严禁推测。"已观察"仅对应可见页面、链接、内容、CTA 或用户材料；公开看到入口不等于已证明该入口的真实贡献，看到产品阶梯不等于已证明用户实际按该路径成交。

## 隐私与安全边界

仅使用用户提供的链路材料与公开可观察信息；不调用未授权付费接口。分析材料仅用于本次分析。
