# 竞品营销策略拆解

## When to Use

- 用户想拆解竞品怎么传播、从哪获客、私域怎么承接、怎么转化成交。
- 用户做增长对标、转化链路参考、钩子与漏斗研究。
- 用户说："XX 怎么获客的""拆解 XX 的转化链路""XX 公域到私域怎么走"。

## When NOT to Use

- 分析竞品定位 / 价值主张 → 用 `competitor-positioning-analysis`（R02）。
- 分析竞品内容栏目与形式 → 用 `competitor-content-analysis`（R03）。
- 诊断"自己"的转化漏斗 → 用 `conversion-funnel-diagnosis`（G02）。
- 设计"自己"的内容获客路径 → 用 `content-acquisition-path`（G01）。

## Inputs

必要信息：

1. 竞品名称与可观察链路（公域账号、落地页、社群、商品页、私信话术）。
2. 分析重点：传播 / 获客 / 承接 / 转化（可多选，默认全链路）。

缺失处理：链路信息不全时，基于可见部分标注「已观察 / 推断 / 未知」；若连公域入口都不明，集中询问一次（仅问竞品名称与可提供的观察材料）。

## Workflow

1. 绘制传播矩阵：竞品在哪些平台发声、各自角色（引流 / 信任 / 转化）；用 Markdown 表格或文本箭头，不强制 Mermaid。
2. 识别可观察获客渠道：列出免费内容、付费投放、搜索、合作等可观察到的渠道；按证据强弱标注「已观察 / 推断 / 未知」，不推测各渠道贡献占比。
3. 还原承接方式：如何把公域用户引到私域 / 社群 / 主页（钩子、入口、话术），标注证据强弱。
4. 梳理转化路径：免费 → 低价 → 正价，或试用 → 付费；关键转化动作，逐节点标注「已观察 / 推断 / 未知」，不将整条路径统一标记为"已观察 / 推断"。
5. 提取关键钩子：什么诱饵（资料 / 优惠 / 诊断）驱动下一步。
6. 总结可借鉴策略（标"建议"）与无法复制的外部条件。
7. 自检：是否含无来源数字、事实与推断是否混淆、是否越界、结论是否可回溯到证据；严禁出现推测的投放占比、转化率或成交数据。

## Output

Markdown 拆解，至少包含：

- 传播矩阵（平台 → 角色，表格或箭头）
- 可观察获客渠道与证据强弱（已观察 / 推断 / 未知，不写占比）
- 承接方式（钩子 + 入口 + 证据标注）
- 转化路径（阶段 + 关键动作，逐节点标已观察 / 推断 / 未知）
- 关键钩子清单
- 可借鉴策略（建议，排序）
- 数据周期与检索日期

## 证据规则

- 优先主动检索：当前信息优先用当前已配置的搜索、浏览器、平台数据或文件分析工具检索公开来源；不默认依赖训练记忆。
- 来源优先级：竞品官方账号、官网、产品页、落地页、公开活动页面和用户提供的一手材料 > 可直接观察的平台内容与数据 > 权威第三方研究或报道 > 主流媒体 > 社交讨论与未经验证的第三方观点。
- 三级标注：区分「已观察事实」「证据支持的推断（标"推断"）」「未知（标"待验证"）」；对投放占比、转化率、成交数据一律标"未知 / 待验证"，不推测。
- 来源冲突：冲突时并列展示各方观点与证据强弱，不强行归一。
- 周期与日期：引用数据须标注数据周期与检索日期。
- 样本说明：给出样本范围、周期、数量与可信度；小样本不输出伪精确比例。

## Boundaries

- 内部投放占比、渠道贡献比例、真实转化率、成交额、ROI 属竞品机密或无法公开观察，一律标"未知 / 待验证"，严禁推测。
- "已观察"仅对应可见页面、链接、内容、CTA 或用户提供的材料；公开看到入口不等于已证明该入口的真实贡献，公开看到产品阶梯不等于已证明用户实际按该路径成交。
- 不编造成交额、ROI 或内部真实运营机制；未知标"待验证"。
- 不替用户设计自身策略；借鉴点标"建议"。
- 转化链路默认用 Markdown 表格或文本箭头（如 公域 → 承接 → 转化），不强制 Mermaid。

## Examples

例（虚构示例，仅示范结构，不代表真实事实；示例中的品牌、路径与数据均为虚构，其中的"已观察 / 推断"仅表示假设案例中的证据状态，非真实品牌事实）：用户输入"拆解某知识付费商家的抖音引流私域路径"。输出：传播（抖音短视频种草，已观察）→ 获客渠道（评论区留资、主页链接，已观察；付费投放，未知）→ 承接（企业微信 + 资料包钩子，已观察）→ 转化路径（主页入口：已观察；9.9 元体验产品：已观察；体验产品是否主要导向正价产品：推断；实际成交路径与转化率：未知）→ 钩子（免费 SOP 资料）→ 借鉴（用资料包做留资钩子，但需自有产品承接）。
