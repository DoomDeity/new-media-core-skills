# 竞品定位分析（competitor-positioning-analysis）

| 项目 | 内容 |
|------|------|
| 中文名称 | 竞品定位分析 |
| 英文名称 | competitor-positioning-analysis |
| Skill 编号 | R02 |
| 当前版本 | 1.3.0 |
| 一句话介绍 | 分析单个竞品服务谁、满足什么需求、价值主张与差异化点，不把推断写成事实。 |

## 解决的问题

想弄清某个竞品到底抢的是哪波人、靠什么立住；做竞品画像、定位对标或找差异化机会时缺结构化方法；担心自己和竞品定位雷同。

## 适合人群

新媒体运营总监 / 操盘手、品牌与市场负责人、内容创作者、需要做竞品对标的人。

## 适用场景

- 弄清某竞品的目标用户、核心需求、价值主张与差异化点。
- 做竞品画像、定位对标、差异化机会识别 / 雷同风险规避。

## 不适用场景

- 分析竞品的内容方向、栏目、形式 → 用 **竞品内容分析（R03）**。
- 拆解竞品的传播、获客、承接、转化路径 → 用 **竞品营销策略拆解（R04）**。
- 研究整个行业格局而非单竞品 → 用 **行业研究（R01）**。
- 设计或诊断"自己"的账号定位 → 用 **账号定位分析（C01）**。

## 核心能力

锁定竞品主体；还原目标用户画像（可推断）；提取核心需求与价值主张；找差异化点（分三类）；写出一句定位陈述；结论—证据对应；在用户提供自身资料时做对比。

## 必要输入

1. 竞品名称（账号 / 品牌 / 产品）。
2. 可获取资料：主页简介、一句话介绍、代表内容或商品、用户评论片段。
3. 分析目的（定位对标 / 差异化机会识别 / 雷同风险规避）。

资料不足时先用可见信息公开推断并标"推断"；连竞品主体都不明则集中询问一次。

## 主要输出

Markdown 分析，至少包含：竞品一句话定位陈述、目标用户画像（场景 / 痛点可推断，人口属性未知标"未知"）、核心需求与价值主张、差异化点（2–3 条）、结论—证据对应表、与自身的对比机会（如用户提供资料）、数据周期与检索日期。

## 简短使用示例

用户说："分析一下 XX 财经科普账号的定位，它抢的是哪波人？"

输出：主体（财经科普创作者）→ 用户（怕被坑的普通消费者 / 职场人，人口属性标"未知"）→ 需求（避险 + 看懂商业套路）→ 价值主张（用通俗表达讲清商业套路）→ 差异化（硬核内容 + 口语化表达，标"推断"）→ 定位陈述 → 结论—证据对应表 → 对比机会（如用户提供自身资料再给）。

## 与相邻 Skill 的区别

R02 只做**单竞品的定位**；R03 做竞品**内容**；R04 做竞品**营销获客转化路径**；R01 看**行业**；C01 设计"自己"的账号定位。分析对象与层级不同。

## 事实、推断与未知的处理规则

三级标注：已观察事实 / 证据支持的推断（标"推断"）/ 未知（标"待验证"）。来源优先级：竞品官方账号、官网、产品页、落地页、公开活动页和用户一手材料 > 可直接观察的平台内容与数据 > 权威第三方研究 > 主流媒体 > 社交讨论。每个结论须对应支撑证据或标"推断 / 未知"。

## 数据、来源或样本限制

用户画像与需求属"推断"，须标注；人口属性（年龄 / 地域 / 收入）无证据标"未知"，不编造。不编造竞品粉丝量、成交额；未知标"待验证"。引用数据须标周期与检索日期。

## 隐私与安全边界

仅使用用户提供的资料与公开可观察信息；不调用未授权付费接口。分析材料仅用于本次分析。
