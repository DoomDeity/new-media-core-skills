# 竞品内容分析（competitor-content-analysis）

| 项目 | 内容 |
|------|------|
| 中文名称 | 竞品内容分析 |
| 英文名称 | competitor-content-analysis |
| Skill 编号 | R03 |
| 当前版本 | 1.3.0 |
| 一句话介绍 | 分析竞品账号的内容方向、栏目结构、形式风格、更新节奏与表现，不编造互动数据。 |

## 解决的问题

想拆解竞品到底发了什么、栏目怎么搭、什么形式、更新多勤、哪些内容表现好；做内容对标、栏目规划参考、爆款规律提取时缺结构。

## 适合人群

新媒体运营总监 / 操盘手、内容创作者、栏目策划、需要做内容对标的人。

## 适用场景

- 拆解竞品发了什么内容、栏目怎么搭、什么形式、更新频率、哪些内容表现好。
- 做内容对标、栏目规划参考、爆款规律提取。

## 不适用场景

- 分析竞品定位 / 价值主张 → 用 **竞品定位分析（R02）**。
- 拆解竞品营销获客转化路径 → 用 **竞品营销策略拆解（R04）**。
- 诊断"自己"账号问题 → 用 **账号诊断（P01）**。
- 规划"自己"的栏目体系 → 用 **内容栏目规划（C02）**。

## 核心能力

盘点内容样本并去重；按三类（常规 / 商业与运营 / 疑似异常）分类统计；栏目归类；主题与形式分布；风格与表达；更新节奏；TOP 内容分析（固定单一口径）；提炼可借鉴 / 需规避点。

## 必要输入

1. 竞品账号名称。
2. 近期内容样本：标题 / 链接 / 形式 / 可选数据（点赞、收藏、评论）；注明样本周期与数量。

仅有账号名时提示用当前已配置工具拉取样本；完全无样本则集中询问一次。

## 主要输出

Markdown 分析，至少包含：栏目结构表、商业与运营内容分类（单独列出，不剔除）、疑似异常内容分类（或"未发现明显异常信号"）、形式与风格概括、更新节奏结论、TOP 内容共性（注明口径与周期）、可借鉴点（建议）、样本范围 / 周期 / 数量与数据来源或"待验证"。

## 简短使用示例

用户说："分析某 PPT 教学账号小红书都发啥。"

输出：栏目（模板领取 / 技巧干货 / 学员案例 / 工具推荐）→ 商业与运营内容（活动内容：抽奖 1 条，单独标注）→ 疑似异常内容（未发现明显异常信号）→ 形式（图文为主）→ 风格（实用、步骤化）→ 节奏（示例：日更，晚 8 点）→ TOP 共性（免费模板钩子 + 步骤清晰，口径：近 90 天收藏量）→ 借鉴（用免费资料做引流钩子）。

## 与相邻 Skill 的区别

R03 只做竞品**内容**拆解；R02 做竞品**定位**；R04 做竞品**营销获客转化路径**；P01 诊断"自己"账号；C02 规划"自己"的栏目。分析对象不同。

## 事实、推断与未知的处理规则

三级标注：已观察事实 / 证据支持的推断（标"推断"）/ 未知（标"待验证"）。来源优先级：竞品官方账号、官网、产品页、落地页、公开活动页和用户一手材料 > 可观察的平台内容与数据 > 权威第三方研究 > 主流媒体 > 社交讨论。小样本不输出伪精确比例，改用"少量样本 / 频次信号"。

## 数据、来源或样本限制

表现结论依赖样本量与数据质量，样本不足标"推断 / 待验证"。不编造互动数据，引用须标来源与周期。TOP 内容排名须固定单一口径（互动量 / 互动率 / 收藏 / 完播 / 转化之一），不同口径不混合排名。跨账号比较不只看绝对互动量，须提醒粉丝规模与发布时间偏差。

## 隐私与安全边界

仅使用用户提供的样本与公开可观察信息；不调用未授权付费接口。分析材料仅用于本次分析。
