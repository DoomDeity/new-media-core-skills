## 主要解决的问题

发现针对品牌 / 产品 / 人物的负面讨论却判断不了风险多大；容易把批评当危机、把未证实指控当事实、把传播风险高等同于指控真实；缺最新状态时依赖旧记忆；不清楚该监测什么信号。

## 3—5 个典型使用场景

1. 评估某话题 / 事件的舆论风险，判断是否需要警惕或监测。
2. 识别负面舆情、争议议题、传播路径、利益相关方与升级风险。
3. 给出监测重点、升级触发条件与转交建议。
4. 区分"反馈信号"与"已观察业务影响"，避免误判危机。

## 适合人群

品牌公关、新媒体运营、危机管理、客服与法务对接、需要做舆论风险评估的人。

## 推荐调用语句

"这个负面会不会闹大，要不要回应？"
"帮我看看这波舆情风险，现在到什么程度了？"
"有哪些信号说明要升级了，我该监测什么？"

## 输入要求

风险相关材料（公开讨论 / 评论 / 媒体 / 用户反馈 / 事件描述）、被讨论主体（品牌 / 产品 / 人物 / 机构）、时间范围与当前进展（可选）、用户关切点。涉及"当前是否升级 / 最近进展"须提供可检索条件或授权实时工具。

## 输出内容

风险研判：事实与未知分离、议题归类、传播路径、利益相关方、五项判断（具体主张证据状态 / 议题类型 / 潜在后果严重程度 / 传播升级风险 / 判断置信度，分别输出）、舆情与反馈信号（与业务影响分离）、已观察业务影响（须可核查）、升级触发条件、转交与建议（不代写声明 / 话术 / 法律结论）。

## 不适用场景

只归纳评论痛点情绪（归 R05）、写单条公开回复（归 P03）、处理具体负面评论（归 P04）、监测平台规则变化（归 P05）、写完整危机公关方案（交人工 / 公关职能）、只判断中性热点趋势（归 R07）。

## 事实和数据边界

以可见公开内容 / 原文 / 可核查数据为依据；传言与未证实指控标未知不重复传播；多家媒体转载同一原始爆料只算 1 原始 + 若干扩散；风险等级低 / 中 / 高附触发因素，证据不足输出"无法判断（研判状态，非风险等级）"；不虚构传播量 / 概率；舆情与反馈信号不自动等于业务影响；传播风险高 ≠ 指控真实；未证实指控不写事实定性。

## 发布文案（适合小红书 / RedSkill 介绍页）

品牌最怕两件事：要么把一条投诉当成天塌了，要么把未证实的爆料当事实去灭火。这个舆情风险识别 Skill 帮你把"要不要紧张"变成五项独立判断：具体主张的证据状态（已证实 / 部分证实 / 未证实 / 无法判断）、议题类型、潜在后果严重程度、传播与升级风险、判断置信度——每一项都分开写、不混成一团。它明确区分"12 条评论"只是反馈信号、不等于已发生的业务影响；传播风险高也不代表指控真实；遇到删不掉的旧闻且实时工具不可用时，会老实说"无法验证当前状态"。它只给监测重点和转交建议，绝不替你写声明或下法律结论。适合公关和运营。
