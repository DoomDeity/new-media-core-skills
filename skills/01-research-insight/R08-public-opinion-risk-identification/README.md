# 舆情风险识别（public-opinion-risk-identification）

| 项目 | 内容 |
|------|------|
| 中文名称 | 舆情风险识别 |
| 英文名称 | public-opinion-risk-identification |
| Skill 编号 | R08 |
| 当前版本 | 1.0.0 |
| 一句话介绍 | 从公开讨论中识别负面舆情、争议与升级风险，给出五项判断，不代写声明、不把传播风险当事实定性。 |

## 解决的问题

发现针对品牌 / 产品 / 人物的负面讨论却判断不了风险多大；容易把批评当危机、把未证实指控当事实、把传播风险高等同于指控真实；缺最新状态时依赖旧记忆下结论；不清楚该监测什么信号、什么时候该转交人工。

## 适合人群

品牌公关、新媒体运营、危机管理、客服与法务对接、需要做舆论风险评估的人。

## 适用场景

- 评估某话题或事件的舆论风险，判断是否需要警惕或监测。
- 识别负面舆情、争议议题、传播路径、利益相关方与潜在升级风险。
- 给出监测重点、升级触发条件与转交建议。

## 不适用场景

- 只归纳一批评论的情绪、意图、高频问题 → 用 `user-comment-analysis`（R05）。
- 撰写对单条评论的公开回复 → 用 `comment-reply`（P03）。
- 分类并处理具体负面评论（如处置话术、流程） → 用 `negative-comment-handling`（P04）。
- 监测平台规则、政策、违规条款变化 → 用 `platform-rule-monitoring`（P05）。
- 撰写完整危机公关方案 / 声明 / 回应稿 → 超出本 Skill 范围，建议交由人工或公关职能处理。
- 只判断中性热点、行业讨论或趋势阶段（无负面争议、无主体受损风险） → 用 `trend-monitoring`（R07）。

## 核心能力

事实梳理与同源去重；议题归类（负面舆情 / 争议 / 单点投诉 / 误会）；传播路径与关键传播者；利益相关方；五项判断（具体主张证据状态 / 议题类型 / 潜在后果严重程度 / 舆情传播与升级风险 / 判断置信度，分别输出、不压缩混写）；综合研判（非机械计数，解释关键驱动因素与限制条件）；升级触发条件；转交与建议（不代写危机声明、回复话术或法律结论）。

## 必要输入

1. 风险相关材料：公开讨论、评论、媒体内容、用户反馈、事件描述（链接 / 截图 / 文本）。
2. 主体信息：被讨论的品牌 / 产品 / 人物 / 机构，及其所在平台与行业。
3. 时间范围与当前已知进展（可选）。
4. 用户关切点（如"会不会影响销量""要不要回应"）。

缺失且可合理假设时，标注"假设"继续；若完全没有可分析材料，集中询问一次（仅问能否提供相关讨论或事件描述）。涉及"当前是否升级 / 最近进展 / 现在风险多大"且需要最新状态时，须实时核验。

## 主要输出

Markdown 风险研判，至少包含：分析对象与材料概况（平台 / 周期 / 匿名化 / 样本量 / 公开私密分层）、时间快照、事实与未知分离、议题归类、传播路径与关键传播者、利益相关方、五项判断（逐项附依据）、舆情与反馈信号（与业务影响分离）、已观察业务影响（须可核查结果、单独列）、关键驱动因素与限制条件、升级触发条件、转交与建议，并明确"未证实主张证据状态标未证实（需继续验证），不构成对任何一方的最终定性；传播升级风险高不等于指控真实"。

## 简短使用示例

用户说："这个负面会不会闹大？"

输出：具体主张证据状态——①"存在 12 条相关评论集中在发货时效"=已证实（可核查事实）；②"品牌实际普遍延迟发货"=未证实（需继续验证）（仅凭评论不能证实普遍延迟）；议题类型（体验）；潜在后果严重程度（低，仅影响个别用户体验）；传播与升级风险（低，单平台、无关键传播者、无跨平台扩散）；判断置信度——对传播升级风险=中、对"品牌普遍延迟发货当前为未证实"这一判定=中（分别标注，不强行给总体置信度）；已观察舆情 / 反馈信号（12 条评论属反馈信号、非业务影响）、已观察业务影响（暂无销量 / 退款 / 取消订单等可核查证据）；建议（客服正常跟进、纳入周报监测、暂不升级，明确"不视为舆情危机"）。

## 与相邻 Skill 的区别

R05 只归纳评论的情绪与高频问题；P03 / P04 写公开回复或处理具体负面评论；P05 监测平台规则变化；R07 看中性热点与趋势阶段。R08 只识别与评估风险，不代写危机声明、回复话术或法律结论；双向边界——负面争议 / 主体受损 / 升级风险归 R08，中性热点 / 行业讨论归 R07，两者输出目标不得混合。

## 事实、推断和未知的处理规则

分层标注：

- **已观察事实**：可见公开内容、原文、可核查数据。
- **证据支持的风险判断（推断）**：标"推断"，须有依据。
- **未知（待验证）**：环境或外部未证实的事项。
- **建议（监测项）**：给用户的行动。

不虚构传播量、影响人数、媒体关注度、升级概率等无来源数字；无可靠口径明确"无法量化"。

## 数据、来源或样本限制

事实优先：以可见公开内容、原文、可核查数据为依据；传言与未证实指控在事实与未知分层中标「未知 / 需继续验证」，不重复传播。来源优先级：原始公开内容 / 官方通报 > 权威媒体报道 > 聚合与社媒讨论 > 单条爆料。来源独立性与同源去重（硬性）：多家媒体转载同一原始爆料只算 1 个原始指控信号 + 若干扩散信号；多个账号复制相同文案 ≠ 多个独立爆料人；追溯原始帖 / 首发时间；同源转载数量不得用于使证据状态更接近"已证实"，但可计入传播范围。定性定级：五项判断分别定性，舆情传播与升级风险等级用低 / 中 / 高并附触发因素；当证据不足以形成等级判断时输出"无法判断（研判状态，非风险等级）"，不当第四档等级。传播风险与业务影响分离：舆情与反馈信号（评论量、投诉数量、转发讨论量、媒体报道、用户反馈、客服咨询量）首先反映反馈 / 传播 / 运营压力，不自动等于业务结果；已观察业务影响须有可核查结果（销量 / 订单 / 退款 / 预约 / 渠道 / 合作 / 转化 / 经核实客服工时成本），单独列；无上述证据时业务影响只能标潜在影响或未知。

## 隐私与安全边界

用户材料含私信、工单、订单、联系方式、非公开账号信息等私人材料，先匿名化且不提交外部工具；已公开的品牌、机构、媒体报道、公开账号和公开帖子，为保证证据回溯可保留必要主体与来源信息，但不得因公开而扩散无关个人敏感信息；未证实指控只做最低必要程度的中性概括，不重复传播煽动性表述。未经用户授权不调用付费数据接口；用户已授权并配置的工具（搜索 / 浏览器 / 平台数据 / 文件分析）可以使用。本 Skill 不承担法律裁定、专业鉴定或最终责任认定。
