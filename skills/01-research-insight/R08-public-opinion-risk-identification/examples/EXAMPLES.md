# 舆情风险识别（public-opinion-risk-identification）使用示例

> 全部示例均为虚构 / 泛化内容，仅用于演示 Skill 的触发、边界与输出结构，不代表任何真实事实。
> 编号：R08。版本：1.0.0。

---

## 例 1：典型正确调用（常规负面不判危机，评论属反馈信号非业务影响）

**输入**：某品牌收到 12 条关于"发货慢"的评论（已匿名化）。

**预期处理逻辑**：

1. 具体主张证据状态：①"存在 12 条相关评论集中在发货时效"=已证实（可核查事实）；②"品牌实际普遍延迟发货"=未证实（需继续验证）（仅凭评论不能证实普遍延迟）。
2. 议题类型：体验。
3. 潜在后果严重程度：低（仅影响个别用户体验，无安全 / 法律 / 财务群体影响；若后续大量同类或官方通报再上调）。
4. 传播与升级风险：低（单平台、无关键传播者、无跨平台扩散）。
5. 判断置信度：对传播升级风险=中（材料完整但样本小）；对"品牌普遍延迟发货当前为未证实"这一判定=中（当前只有评论材料，没有订单 / 物流 / 履约数据，既不能证实也不能推翻，故判定为未证实需继续验证）；分别标注，不强行给总体置信度。
6. 已观察舆情 / 反馈信号：12 条评论属反馈信号，非业务影响；已观察业务影响：暂无销量 / 退款 / 取消订单等可核查证据。
7. 建议：客服正常跟进、纳入周报监测、暂不升级，明确"不视为舆情危机"。

**预期输出结构**：五项判断逐项附依据、舆情与反馈信号（与业务影响分离）、已观察业务影响（无则标未知）、升级触发条件、转交与建议。

**标注示例**：证据状态【已证实】/【未证实（需继续验证）】；风险【低】；置信度分对象标注；业务影响【暂无已观察证据】。

## 例 2：典型正确调用（未证实指控，来源真实性与传播风险分别判断）

**输入**：某产品被 1 个账号发帖称"有安全隐患"，2 个平台出现讨论（1 平台为原帖转发、1 平台为独立讨论），1 个行业 KOL 转发，无权威检测报告。

**预期处理逻辑**：

1. 具体主张证据状态：①"某账号发布了安全指控"这一发帖行为可见=已证实（事实）；②"产品确有安全问题"=未证实（需继续验证），不重复为事实、不对品牌作无证据定性。
2. 议题类型：安全。
3. 潜在后果严重程度：高，但须说明成立条件——仅"若安全隐患属实"时潜在后果为高；当前未证实，不得写成后果已经发生。
4. 传播与升级风险：中（当前跨 2 平台讨论 + 1 KOL 转发，但无权威检测报告、无媒体或机构跟进；同源转载仅支持传播范围，不能单独上调风险等级；若后续权威媒体 / 机构介入或持续增长再上调）。来源真实性与传播风险分别判断（发帖真实 ≠ 产品确有安全问题）。
5. 判断置信度：对传播风险=中（可见材料较充分但缺权威证据）；对"该主张当前为未证实"这一判定=中（当前无检测报告或权威结论，可确定未被证实但无法排除材料之外另有证据，故不写高）；分别标注。
6. 关键驱动因素：跨平台扩散与 KOL 转发构成传播信号，但权威证据与持续新增不足，风险暂定中；限制条件：安全隐患本身未证实，风险等级随权威媒体 / 机构介入或持续增长可上调。
7. 建议：先采集证据、准备事实澄清，暂不公开定性；交由人工决定是否回应。

**预期输出结构**：同例 1；明确"传播风险即便上调为高也不代表安全指控真实"；升级触发条件（权威媒体跟进 / 官方通报 / 出现人身伤害案例）。

## 例 3：典型正确调用（公开材料与私密投诉的隐私边界）

**输入**：用户提供两类材料——(a) 公开的品牌官方微博下 30 条用户投诉评论（公开帖子，可保留品牌名与帖子链接用于证据回溯）；(b) 用户私信截图含真实姓名、手机号与订单号（私人材料）。

**预期处理逻辑**：公开投诉评论直接分析并保留品牌与帖子链接（匿名化其他提及的个人昵称即可）；私信材料先匿名化为"用户 X"且不提交外部工具，仅用于说明"存在私下集中投诉"这一事实，不暴露姓名 / 手机号 / 订单号；整体未因"公开"而扩散无关个人敏感信息，未证实内容仅做中性概括。

**预期输出结构**：同例 1；隐私分层说明（公开材料保留来源、私密材料匿名化不提交外部）。

## 例 4：必须追问

**输入**："帮我评估一下这个舆情风险。"（未提供任何讨论、事件描述或主体信息）。

**预期处理逻辑**：完全无材料却要求给风险等级，会实质影响判断，集中询问一次："请提供相关讨论或事件描述（链接 / 截图 / 文本），以及被讨论的品牌 / 产品 / 人物 / 机构。"

**预期输出结构**：不产出风险研判，仅给出一条追问；获得材料后再按标准流程执行。

## 例 5：不应调用本 Skill（反例）

**输入**："帮我写一条回复，回击那个说我们发货慢的用户。"

**预期处理逻辑**：这是撰写对单条评论的公开回复，不属于舆情风险识别。应转交 **评论区回复（P03）** 或 **负面评论处理（P04）**；R08 只评估风险、给出监测与转交建议，不代写回复话术或声明。

**预期输出结构**：不产出回复文案；说明边界后建议转 P03 / P04，或等待用户确认。

## 禁止扩大职责提示

- 不代写危机声明 / 评论回复 / 法律结论；未证实前不对个人或机构作负面定性。
- 不把批评 / 投诉 / 负面情绪自动定义为舆情危机；证据不足归为常规反馈或待观察。
- 五项判断分别输出（具体主张证据状态 / 议题类型 / 潜在后果严重程度 / 传播升级风险 / 判断置信度），不压缩混写为单一结论。
- 舆情与反馈信号（评论量、投诉数量等）不自动等于业务结果；已观察业务影响须有可核查证据。
- 传播风险高 ≠ 指控真实；"待观察"仅作后续监测状态，不写入证据状态 / 风险等级 / 判断置信度字段。
