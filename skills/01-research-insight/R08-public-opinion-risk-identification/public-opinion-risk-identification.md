# 舆情风险识别

## When to Use

- 用户发现针对品牌 / 产品 / 人物 / 机构的负面讨论、争议话题或事件，想判断风险有多大。
- 用户要评估某条内容或话题是否会升级、需要监测什么、什么情况下该警惕。
- 用户说："这个负面会不会闹大""帮我看看这波舆情风险""这个争议要不要重视""哪些信号说明要升级了"。

## When NOT to Use

- 只归纳一批评论的情绪、意图、高频问题 → 用 `user-comment-analysis`（R05）。
- 撰写对单条评论的公开回复 → 用 `comment-reply`（P03）。
- 分类并处理具体负面评论（如处置话术、流程） → 用 `negative-comment-handling`（P04）。
- 监测平台规则、政策、违规条款变化 → 用 `platform-rule-monitoring`（P05）。
- 撰写完整危机公关方案 / 声明 / 回应稿 → 超出本 Skill 范围，建议交由人工或公关职能处理。

## Inputs

必要信息：

1. 风险相关材料：公开讨论、评论、媒体内容、用户反馈、事件描述（链接 / 截图 / 文本）。
2. 主体信息：被讨论的品牌 / 产品 / 人物 / 机构，及其所在平台与行业。
3. 时间范围与当前已知进展（可选）。
4. 用户关切点（如"会不会影响销量""要不要回应"）。

缺失处理：主体或材料缺失且可合理假设时，标注"假设"继续；若完全没有可分析材料，集中询问一次（仅问能否提供相关讨论或事件描述）。

## 缺失信息处理

- 有材料但缺主体背景：标"假设"继续，风险判断标注适用范围。
- 材料仅为少量负面评论：先按"常规负面反馈"归类，不自动升级为危机；说明证据不足。
- 未证实的指控：列为「未知 / 待验证」，不重复传播为事实，不对个人或机构作无证据定性。
- 会实质影响风险判断（如完全无材料却要求给风险等级）：集中询问一次。

## Workflow

1. 隐私匿名化：材料含姓名、联系方式、账号 ID 等可识别信息先匿名化；未匿名化材料不提交外部检索 / 第三方工具。
2. 事实梳理：区分「已观察事实」（可见的公开内容、讨论、媒体原文）与「未知」（未证实部分），不把传言当事实。
3. 议题归类：负面舆情 / 争议议题 / 单点投诉 / 误会澄清，分别归集。
4. 传播路径：识别首发平台、扩散平台、关键传播者参与情况（标注「事实」或「推断」）。
5. 利益相关方：列出受影响方、发声方、可能放大的第三方（如媒体 / KOL）。
6. 风险定级：按下方「风险等级依据」逐项打勾，给出定性等级（低 / 中 / 高 / 待观察）并说明依据；不使用无依据的精确概率（如"升级概率 73%"）。
7. 升级触发条件：列出后续应重点监测的信号（如跨平台扩散、媒体跟进、关键传播者入场）。
8. 给用户的建议：给出监测重点与应对时机建议，标「建议」；**不代写危机声明或回复话术**。
9. 自检：是否把批评当危机？是否虚构传播量 / 概率？未证实指控是否标清？是否越界代写声明？

## Output

Markdown 风险研判，至少包含：

- 分析对象与材料概况（平台 / 周期 / 匿名化说明 / 样本量）
- 事实与未知分离（已观察事实 vs 待验证，不混写）
- 议题归类（负面舆情 / 争议 / 单点投诉 / 误会）
- 传播路径与关键传播者（事实 / 推断标注）
- 利益相关方
- 风险等级与依据（逐条列出触发的因素）
- 升级触发条件（监测清单）
- 给用户的建议（标「建议」；监测重点与时机，不代写声明 / 话术）
- 明确"本研判基于所提供材料，未证实内容标为待验证，不构成对任何一方的最终定性"

如需落盘，文件名如 `舆情风险识别-主体-YYYYMMDD.md`。

## Evidence / Quality Rules

- 事实优先：以可见公开内容、原文、可核查数据为依据；传言与未证实指控标「未知 / 待验证」，不重复传播。
- 来源优先级：原始公开内容 / 官方通报 > 权威媒体报道 > 聚合与社媒讨论 > 单条爆料。低优先级须降级标注。
- 三级 + 一类标注：区分「已观察事实」「证据支持的风险判断（推断）」「未知（待验证）」「建议（监测项）」。
- 禁止虚构：不编造传播量、影响人数、媒体关注度、升级概率等无来源数字；无可靠口径明确"无法量化"。
- 定性定级：风险等级用低 / 中 / 高 / 待观察，必须附触发因素，不凭感觉给分。
- 不越界定性：未证实前不对个人或机构作负面定性；如证据增强再更新判断并标注变化。

## Boundaries

- 只识别与评估风险，不负责完整危机公关方案、声明撰写或评论回复（那是人工 / P03 / P04 职责）。
- 不把批评、投诉或负面情绪自动定义为舆情危机；证据不足时归为常规反馈或待观察。
- 风险等级必须基于明确因素（见下），不使用无依据精确分数或概率。
- 不虚构传播规模、影响人数、媒体关注度或升级概率。
- 未证实指控不写成事实，不对个人或机构作无证据定性。
- 输出可给监测重点与升级触发条件，但不得越界代写危机声明或处理话术。
- 用户材料含个人信息须匿名化，不提交未经处理的隐私材料到外部工具。

### 风险等级依据（逐项勾选）

- 传播范围（单平台少量 / 多平台 / 广泛）
- 增长速度（平稳 / 加速，标事实或推断）
- 议题严重性（体验问题 / 安全 / 法律 / 伦理 / 群体利益）
- 证据真实性（已证实 / 部分证实 / 未证实）
- 关键传播者参与（普通用户 / 媒体 / KOL / 机构账号）
- 跨平台扩散（单一 / 跨平台）
- 是否涉及安全、法律、伦理或群体利益

触发越多、严重度越高，等级越高；须逐条说明。

## Examples

例 1（虚构示例，仅示范结构，不代表真实事实）：某品牌收到 12 条关于"发货慢"的评论（已匿名化）。输出：事实（12 条集中在发货时效，标事实）→ 议题（单点投诉 / 常规负面反馈）→ 传播（单平台、无媒体跟进，标事实）→ 利益相关方（下单用户）→ 风险等级（低，依据：传播范围单平台、议题为体验问题、无关键传播者、无跨平台）→ 建议（标建议：客服正常跟进，纳入周报监测，暂不升级）→ 明确"不视为舆情危机"。

例 2（虚构示例，未证实指控）：某产品被一账号发帖称"有安全隐患"，2 个平台出现讨论，1 个行业 KOL 转发，但无权威检测报告。输出：事实（发帖与转发可见，标事实）→ 未知（安全隐患未证实，标待验证，不重复为事实，不对品牌作无证据定性）→ 传播（跨 2 平台、KOL 参与，标事实；增速待观察，标推断）→ 风险等级（高，依据：跨平台扩散 + 关键传播者 + 涉及安全议题，但证据真实性为"未证实"）→ 升级触发条件（出现权威媒体跟进 / 官方通报 / 出现人身伤害案例）→ 建议（标建议：先采集证据、准备事实澄清，暂不公开定性；交由人工决定是否回应）。

## 最终自检

- 触发准确：仅覆盖舆情风险识别与评估，已互指 R05 / P03 / P04 / P05。
- 边界清晰：不代写声明 / 话术，不把批评当危机，不处理具体评论回复。
- 无来源数字：未编造传播量 / 概率；示例均为虚构且自洽（12 条、2 平台等）。
- 未证实指控已标清：例 2 明确"待验证"，不作无证据定性。
- 风险等级有依据：两例均逐条列出触发因素。
- 推断已标注：传播增速、风险推断均标「推断」。
- 未写死工具：仅用"当前已配置的搜索、浏览器、平台数据或文件分析工具"。
- 未越界：只给监测重点与建议，不代写公关声明。
